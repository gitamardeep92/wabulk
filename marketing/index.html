FAQ

Common questions

Do I need a Meta Business account or BSP approval?

No. WaBulk works by connecting your existing WhatsApp Business number directly via QR scan. You don't need any Meta approval, BSP partnership, or phone number verification process — just scan once and start sending.

What kind of messages can I send?

WaBulk is built strictly for informational and transactional notifications — order confirmations, membership reminders, payment receipts, appointment alerts, etc. Promotional or spam content is not permitted and will result in account suspension.

Will my WhatsApp account get banned?

We minimize this risk by enforcing humanized delays between messages (2–8 seconds), preventing duplicate sends, and limiting daily send volumes. Accounts are safest when sending legitimate transactional content to existing customers who expect messages from you.

How do dynamic variables work?

Use {{variable_name}} anywhere in your template. Pass the values per recipient in the vars object when calling the API. We replace every placeholder before sending — so each recipient gets a fully personalized message.

Can I use this from any programming language?

Yes. WaBulk is a plain REST API with JSON. Any language that can make HTTP requests works — Node.js, Python, PHP, Ruby, Go, Java, no-code tools like Zapier and Make, and more. We provide code examples for the most popular environments.

What happens if I hit my monthly limit?

The API returns a 429 error with a clear message. Queued messages beyond the limit are skipped. You can upgrade your plan at any time during the month and the new limit applies immediately.
